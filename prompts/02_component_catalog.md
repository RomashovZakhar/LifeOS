# 02 — Component catalog

Используя только `analysis/01_reverse_engineering.md` и скриншоты,

определи каждый повторно используемый UI-компонент эталона.

## Для каждого компонента опиши

- назначение
- анатомию
- иерархию
- варианты
- состояния
- интервалы
- взаимодействие
- доступность
- возможности повторного использования

## Правила

- Не изобретай компоненты.
- Опиши только то, что реально видно на скриншотах.
- Если для состояний нет скриншотов — попроси предоставить.

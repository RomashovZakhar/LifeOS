## Правила

1. Ничего не перепроектировать. Не «улучшать UX» продуктово. Не добавлять свои продуктовые идеи.
2. Если реализация требует предположения — стоп и вопрос.
3. Никогда не менять дизайн молча.
4. Внедрять **по одному экрану** (или согласованному вертикальному срезу) за раз.
5. После каждого экрана — короткое self-review и ждать одобрения.

## UI fidelity (обязательно)

Заказчик — **дизайнер**. Визуальное качество — first-class требование, не «потом полируем».

1. Ориентир: `analysis/screenshots/` + `03_design_system.md` + модульные спеки. Сверять экран со скрином эталона, не с абстрактной схемой.
2. Плотность, типографика (sans/mono), радиусы, weekend banding, accent badge, dock — как в эталоне; токены 03, не «дефолтный Vue UI».
3. Заглушки Phase 0 не считать UI. Каждый пользовательский экран сдавать в состоянии, готовом к дизайн-ревью.
4. Self-review обязан включать: отступы, выравнивание сетки, контраст secondary, hit targets dock, dark **и** light.

## Self-review после экрана

Искать:

- несоответствия утверждённому дизайну
- дублированный код
- ненужные абстракции
- архитектурные запахи
- проблемы с maintainability
- регрессии offline / PWA

Сначала зафиксировать fidelity экрана. Рефакторинг без смены UX — только после апрува экрана, отдельным мини-шагом.

# 04 — Information architecture

Теперь, когда эталон разобран, спроектируй информационную архитектуру **LifeOS**.

## Вход

- `docs/PRODUCT.md`, `VISION.md`, `ANTI_GOALS.md`, `TECH.md`
- `analysis/01_reverse_engineering.md`
- `analysis/02_component_catalog.md`
- `analysis/03_design_system.md`

## V1 scope (обязательно все три)

- Habits
- Workouts
- Morning

## Цель

Не изобретать новый язык дизайна.

Расширить существующий язык эталона так, чтобы три модуля ощущались одним приложением.

## Правила

- Прежде чем предлагать экран — проверь, можно ли переиспользовать существующий interaction pattern.
- Каждый раз, когда вводишь новое взаимодействие — обоснуй, почему reuse невозможен.
- Habits — полноценный модуль LifeOS, не «просто клон эталона без спецификации».
- Задавай вопросы, прежде чем принимать продуктовые решения.
- Учти offline и установку на iOS Home Screen (навигация и состояния без сети).

## Выход

`analysis/04_information_architecture.md`

Включи: карта экранов, навигация между модулями, ключевые user flows для каждого модуля, что переиспользуется из эталона.

Стоп. Жди одобрения.

# 05 — Habits module

Выступай как product designer эталонного продукта, расширяющий его для LifeOS.

## Задача

Спроектировать полноценный модуль **Habits** для LifeOS.

Это обязательный модуль V1 — не «эталон как есть без спецификации» и не заглушка.

## Вход

- утверждённые `analysis/01` … `04`, `04b`
- `docs/PRODUCT.md`, `ANTI_GOALS.md`, `VISION.md`

## Ограничения

- Повторно использовать визуальный язык эталона.
- Повторно использовать существующие interaction patterns, spacing, навигационные идеи.
- Избегать новых UI-паттернов без крайней необходимости; если нужны — обосновать.
- Не копировать подписку, маркетинг и бренд эталона.
- Оптимизировать для скорости, спокойствия, одной руки, offline.

## Опиши подробно

- каждый экран
- каждое взаимодействие
- крайние случаи
- пустые состояния
- создание / редактирование / удаление
- историю / обзор во времени
- progressive disclosure
- как Habits стыкуется с Workouts и Morning на уровне IA (без поглощения их scope)

## Правила

- Не писать код.
- Если несколько UX-решений — плюсы/минусы, затем вопросы.
- Не решать продуктовые развилки молча.

# Data Model

## Status

Модель данных V1 **утверждена**.

Источник правды:

**`analysis/04b_data_model.md`** (секция «Утверждено» + тело схемы).

Кратко: IndexedDB; trackers + entries; checklist snapshots; workout sessions (длительность портала = проекция сессии); export JSON.

## Process (как было спроектировано)

1. Агент предложил итоговую схему под утверждённую IA (hub + portals).
2. Пользователь утвердил; согласованы уточнения IA / PRODUCT.
3. Изменения схемы дальше — только явно, с обновлением `04b` и вопросом, если ломается совместимость или Export.

## Hard constraints

- Один пользователь, только локальные данные.
- Offline-first: схема работает без сети.
- Три возможности V1: **Habits**, **Workouts**, **Checklist** (ex-Morning).
- Общие примитивы (день, tracker) при одном корне Habits; не три изолированных приложения в store без связей.
- Не оптимизировать схему под Strong / Apple Health и т.д.

## After approval

Любое изменение схемы — только явно, с обновлением `analysis/04b_data_model.md` и вопросом пользователю, если ломается совместимость.

# Процесс проектирования

Порядок задан оркестратором (`ORCHESTRATOR.md`). Кратко:

1. Разобрать эталонное приложение.
2. Извлечь компоненты и язык дизайна.
3. Собрать design system (токены, правила).
4. Спроектировать информационную архитектуру LifeOS (habits + workouts + checklist).
5. Предложить и утвердить модель данных.
6. Спроектировать каждый модуль V1 на языке эталона.
7. Проверить согласованность.
8. Только после этого писать код.

На каждом этапе лучше задавать уточняющие вопросы, чем делать предположения.

Если не уверены — остановитесь и спросите.

Артефакты каждого шага сохраняются в `analysis/`.
